CLI Reference

The Jenkins workflow, short enough to keep open while practicing.

Use this page while working through the hands-on AWS labs. It keeps the important commands close without digging through the full notes.

Workflow

Standard command loop

01

systemctl start jenkins

Start the Jenkins service on Linux.

->
02

cat initialAdminPassword

Get the initial admin password from /var/lib/jenkins/secrets.

->
03

Jenkins CLI

Use java -jar jenkins-cli.jar to interact via command line.

->
04

Trigger Build

Trigger a job remotely using curl POST with API token.

->
05

Pipeline Syntax

Use the Jenkins Pipeline Syntax generator for steps.

->
06

Clean Workspace

Use cleanWs() in the post block to clear the workspace.

->
Daily Jenkins commands 
sudo systemctl status jenkins
sudo systemctl enable jenkins
cat /var/lib/jenkins/secrets/initialAdminPassword
java -jar jenkins-cli.jar -s http://localhost:8080/ help
curl -X POST http://localhost:8080/job/MyJob/build
Situation Command Use carefully because
Check Service Status systemctl status jenkins Check if Jenkins is running
Restart Jenkins systemctl restart jenkins Applies new plugin configurations
Tail Logs journalctl -u jenkins -f Watch Jenkins logs in real-time
Trigger Build via CLI jenkins-cli build my-job Requires authentication
Safe Restart URL http://server/safeRestart Waits for builds to finish before restart
